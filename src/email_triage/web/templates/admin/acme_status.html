WebAuthn fields apply to all postures — they configure hardware-key authentication, not the cert itself.

ACME (Let's Encrypt)

LE-side validation resilience

After our public-resolver gate passes, LE still queries the record from its own recursive chain. Negative caches and slow downstream resolvers can fail the first try. Match acme.sh: pause briefly to let LE-side caches expire, then retry the full order on ValidationError.

DNS-01 (RFC-2136 dynamic DNS update){{ m.help("DNS-01 proves domain control by writing a TXT record at _acme-challenge.. This implementation uses RFC-2136 dynamic update -- the renewer signs an UPDATE message with a TSIG key and sends it directly to the authoritative nameserver. No HTTP-01 (would need port 80 exposed); no DNS-provider plugins (BIND-native).") }}

Public-DNS propagation gate

After the TXT is visible on your authoritative server, the renewer waits for these public recursive resolvers to see it before signaling Let's Encrypt. Catches the propagation window between an internal update target and the public-facing nameservers. Matches acme.sh behavior (multi-resolver, long retry).
