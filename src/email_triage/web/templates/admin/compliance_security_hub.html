Compliance & Security

Six related controls grouped here. HIPAA mode + session policy + brute-force protection + CSRF posture + classifier BAA gate + audit-chain integrity all surface in one place so an auditor walking the install doesn't have to context-switch across two or three pages.

🏥 HIPAA mode

System-wide flag + per-account stacking. Per-account HIPAA flag uses sticky inheritance from system flag at account creation time. PHI scrubbing in logs + notifications + escalation alerts is mode-aware.

HIPAA mode
⏱️ Session policy

session_ttl_secs for non-HIPAA installs (default 24h), hipaa_session_ttl_secs as the HIPAA-mode cap (default 15 min, hard ceiling 30 min). §164.312(a)(2)(iii) Automatic Logoff.

Session TTL
🛡️ CSRF protection

Soft-launch counter + 24h/7d rejects rate + top-N offending paths. Operator flips csrf_enforce once 24h count is steady at zero. Operator-defined exempt prefixes for custom integrations.

CSRF + exempt prefixes
🔐 Login rate limits

Per-email + per-IP rolling-window failure counters guarding OTP / WebAuthn / dev-keypair / passkey login surfaces. Source of truth: auth_events table. max=0 per scope is the kill switch. §164.312(a)(2)(i) Unique User Identification.

Login limits
📜 Classifier BAA gate

HIPAA-flagged accounts refuse to classify when the configured LLM endpoint is non-local and no Business Associate Agreement acknowledgment is on file. Per-(backend, host) ack persisted via the privacy chip on /config.

BAA + boundary events
🔗 Audit chain integrity

Every log_entries row carries a hash of its predecessor. Tampering breaks the chain. The compliance page surfaces the current chain status and recent boundary events; an external verifier CLI is available via verify_log_chain().
