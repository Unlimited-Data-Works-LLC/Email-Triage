Lives outside the IMAP connection fieldset (host/port/user/pass) because folder selection is a watch / ingestion concern, not a connection concern, and the operator review feedback was that the legacy placement on Provider + Auth crossed those wires. IMAP IDLE is per-mailbox, per-connection — watching N folders means N concurrent IDLE connections. When ``folder_options`` is populated (live discovery succeeded for an existing account), we render a checkbox list so operators can pick Spam + Sent + custom folders in one click. When discovery fails or the account is new and not yet authenticated, we fall back to a comma- separated text input so the form is never stuck behind an IMAP probe. INBOX is always a safe default. The save handler reads either ``mailboxes`` (multi-checkbox) or ``mailboxes_csv`` regardless of which tab the field came from — both ride the same form submit. #}
